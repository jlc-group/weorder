# กฎการทำงานที่ต้องปฏิบัติตาม

## ก่อนบอกว่า "ข้อมูลถูกต้อง" ต้องทำสิ่งนี้ก่อน:

1. **เปรียบเทียบกับวันก่อนหน้า/หลัง**
   - ถ้าตัวเลขต่างกันมากกว่า 50% ต้องตั้งคำถามทันที
   - ไม่บอกว่า "ถูกต้อง" จนกว่าจะตรวจสอบแล้ว

2. **ตรวจสอบ Sync Status**
   - เช็คว่ามี Sync Job ที่ล้มเหลวหรือค้างอยู่ไหม
   - ดู `platform_config.last_sync_at` ว่าอัปเดตล่าสุดเมื่อไหร่

3. **ถ้าข้อมูลดูผิดปกติ ต้องบอกผู้ใช้ทันที**
   - ไม่บอกว่า "ถูกต้อง" แล้วค่อยมาพบปัญหาทีหลัง
   - ยอมรับความไม่แน่ใจและเสนอทางตรวจสอบเพิ่ม

## ก่อน Re-sync ข้อมูล:

1. **บันทึกจำนวนเดิมก่อน** - เพื่อเทียบกับหลัง sync
2. **ใช้ช่วงเวลาที่ถูกต้อง** - ตรวจสอบ timezone (Asia/Bangkok)
3. **รายงานผลต่าง** - บอกว่าเพิ่มขึ้นกี่รายการ

## หลังทำงานเสร็จ:

1. **Verify ด้วย Query ตรง** - ไม่เชื่อ API Response อย่างเดียว
2. **แสดงหลักฐานที่จับต้องได้** - ตัวเลขก่อน/หลัง
3. **ยอมรับผิดถ้าทำพลาด** - ไม่แก้ตัว
